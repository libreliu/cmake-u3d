IFXRESULT CIFXModifierChain::AddModifier (  IFXModifier &  rInModifier,
U32  index,
BOOL  isReqValidation
)  [virtual]
 

Appends the incoming Modifier at the specified index.

Parameters:
rInModifier The incoming Modifier that will be added.
index The index where the modifier is to be inserted defaults to the end of the modifier chain. Otherwise the parameter must be < Number of modifiers in the modifier chain. If isReqValidation is FALSE, then the index can be past the end of the chain and NULL modifier entries will be inserted as place holders. If the current modifier at the index is NULL, the entry will be replaced with the new modifier.
isReqValidation Defines if incomming Modifier requires validation or no. If this is TRUE, the modifier chain will return an error code. This occurs when a modifier cannot be enabled or if any other modifier moves to a disabled state. If this parameter is FALSE then the operation will succeed unless a modifier in the chain returns an error code during the validation process.
Precondition:
The dependencies of rInModifier is satisfied by the resultant ModifierChain DataPacket.
Returns:
One of the following IFXRESULT codes:

  • IFX_OK
    No error.

  • IFX_E_MODIFIERCHAIN_INCOMPATIBLE_MODIFIER
    The specified Modifier was NOT added because the resultant DataPacket at the end of the chain did not satisfy the input requirements specified by rInModifier.

  • IFX_E_OUT_OF_MEMORY
    The Modifier was not appended because not enough memory was available.

Implements IFXModifierChain.

IFXRESULT CIFXModifierChain::ApplyNewModifierState (  IFXModifierChainState *  pInNewState  )  [private]
 

Takes a New Modifier Chain State and attempts to set it as the active modifier chain state.

Side effects will delete pInNewState if application fails, if application succeeds m_pOldState will hold the last valid state of the mod chain. Which must either be directly deleted via call to ClearOldState or RestoreOldState. This call whether it succeeds or fails clears the old and cached states off of the appended chain hier.

Parameters:
pInModifierChain A pointer to the incoming ModifierChain that will precede the ModifierChain. May be Null, to imply that there should be no prepended chain after this call succeeds.
Precondition:
The first Modifier in the ModifierChain is compatible with the resultant DataPacket of pInModifierChain.
Postcondition:
The resultant DataPacket contents from pInModifierChain is treated as input to the first Modifier in the ModifierChain.
Returns:

  • IFX_OK
    No error.

    The ModifierChain was NOT prepended because not enough memory was available.

  • IFX_E_NOT_INITIALIZED

IFXRESULT CIFXModifierChain::Update (  IFXSubject *  pInSubject,
U32  changeBits,
IFXREFIID  rIType
)  [virtual]
 

Notification that the provided subject has changed.

Parameters:
pInSubject A pointer to the IFXSubject interface of the component that changed.
uInChangeBits The aspects of interest that changed as defined by the supplied IFXSubject's published interest bits. if uInChangeBits are 0, then the subject is shutting down, and should be released if AddRef'ed, or local references should be cleared in the weak reference case.
Returns:
One of the following IFXRESULT codes:

  • IFX_OK
    No error.

  • IFX_E_INVALID_POINTER
    pInSubject was NULL.

  • IFX_E_OUT_OF_MEMORY
    Not enough memory is available to resolve the change.
Note:
This method is typically called from IFXSubject::PostChanges().

This method with uInChangeBits = 0 allows for the circular reference between the Observer and the Subject to be broken.
