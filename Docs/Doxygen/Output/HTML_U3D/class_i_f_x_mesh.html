Detailed Description

This object represents a single mesh (3D surface).

The surface contains indexed triangle lists. The triangle vertices contain a variable number of vertex attributes that are specified upon allocation. These attributes can include positions, normals, colors, and texture coordinates. The triangles are defined by an array of triangle faces, with each face containing three vertex indices. These indices will be 32 bits in size if the number of allocated vertices are more than 65536, otherwise the indices will be 16 bits in size.

The mesh is divided into two or more distinct memory pools: vertices and faces. There could also be additional vertex pools for storing texture coordinates depending upon the settings of the IFXVertexAttributes object passed in during IFXMesh::Allocate(). These pools of memory are contained in IFXInterleavedData objects.

virtual IFXRESULT IFXAPI IFXMesh::Allocate (  IFXVertexAttributes  vaAttribs,
U32  uNumVertices,
U32  uNumFaces
)  [pure virtual]
 

Allocates (or reallocates) memory for the vertex and face pools depending on the vertex attributes and the number of vertices and faces that were passed in.

Parameters:
vaAttribs Input IFXVertexAttributes object that specifies what data is stored in each vertex.
uNumVertices Input U32 specifying how many vertices to allocate space for. The size of each vertex is dependent on the settings of vaAttribs.
uNumFaces Input U32 specifying the number of faces to allocate space for. The size of each face is dependent on how many vertices are being allocated. If there are less than 65536 vertices, each face is 6 bytes, otherwise each face is 12 bytes.
Returns:
IFXRESULT.
Return values:
IFX_OK No error.
IFX_E_OUT_OF_MEMORY There is not enough memory available to complete this method. The mesh is in an undefined state when this occurs.

virtual IFXRESULT IFXAPI IFXMesh::GetVectorIter (  IFXenum  eType,
IFXIterator &  iter
)  [pure virtual]
 

Retrieves a vector iterator that will iterate over a portion of the vertex data associated with the mesh.

This method will fail if the iterator type (say IFXVector4Iter) that is passed in is bigger than the data it is iterating over (for example, Positions, which are IFXVector3 in size).

Parameters:
eType Input IFXenum value that specifies what the iterator should iterate over. Possible values are:
  • IFX_MESH_POSITION: Iterate over vertex positions.
  • IFX_MESH_NORMAL: Iterate over vertex normals.
  • IFX_MESH_DIFFUSE_COLOR: Iterate over vertex diffuse colors.
  • IFX_MESH_SPECULAR_COLOR: Iterate over vertex specular colors.
  • IFX_MESH_TC0: Iterate over texture coordinate layer 0.
  • IFX_MESH_TC1: Iterate over texture coordinate layer 1.
  • IFX_MESH_TC2: Iterate over texture coordinate layer 2.
  • IFX_MESH_TC3: Iterate over texture coordinate layer 3.
  • IFX_MESH_TC4: Iterate over texture coordinate layer 4.
  • IFX_MESH_TC5: Iterate over texture coordinate layer 5.
  • IFX_MESH_TC6: Iterate over texture coordinate layer 6.
  • IFX_MESH_TC7: Iterate over texture coordinate layer 7.
iter Output IFXIterator reference that will be set up to iterate over the data specified with eType.
Returns:
IFXRESULT.
Return values:
IFX_OK No error.
IFX_E_INVALID_RANGE Either the attribute specified by eType does not exist in the mesh, or the passed in iter is too big for the data element that would be iterated over.

Increases the size of the MaxNumberVertices and MaxNumberFaces.

If necessary, new vertex and face memory pools will be allocated. All data contained in the mesh will be preserved.

If the current number of vertices is less than 65536 and the new number of vertices is greater than 65536, the faces will be converted from 16 bit indices to 32 bit indices. In this case, the index numbers that currently exist will be preserved.

virtual IFXRESULT IFXAPI IFXMesh::TransferData (  IFXMesh &  rSrcMesh,
IFXMeshAttributes  uTransferAttr,
IFXMeshAttributes  uCopyAttr
)  [pure virtual]
 

Conditionally duplicates the source mesh, copying some interleaved data elements and referencing others.

This allows meshes to be copied quickly, with only the data arrays that are going to be changed being duplicated. All the other data will be referenced from the source mesh. If an attribute that is present in the source mesh is not included in the Transfer or the Copy Attr Bits, it will not be included in the new mesh.

Parameters:
rSrcMesh Input reference to an IFXMesh containing mesh data to be copied into this mesh.
uTransferAttr Bits representing the attributes that should be referenced from the Src Mesh. Specifing bits that are not present in Src mesh has no effect.
uCopyAttr Bits representing the attributes of the source mesh that should be copied and not just referenced from the source mesh. Specifying bits that reflect attributes not present in the source mesh will cause the new attribute arrays to be created. In this case, Texture coordinates inserted this way will default to 2D.
Returns:
IFXRESULT.
Return values:
IFX_OK No error.
IFX_E_INVALID_POINTER The vertex attributes of rSrcMesh are not compatible with this mesh.

Implemented in CIFXMesh.
