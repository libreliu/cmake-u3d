Dynamic Library Packaging Diagrams

This page contains high-level packaging diagrams for the U3D dynamic libraries.

Many key functionalities and components are represented in each of these diagrams, however not all of them are shown.

Core DL

The U3D Core dynamic library contains the IFXCOM cross-platform component object model and common U3D functionality used by most clients that use the U3D base profile (or also the extensible profile when a future edition of the U3D specification is implemented).

Exporting DL

The U3D Exporting dynamic library contains IFXCOM plug-ins and services to prepare and encode geometry to U3D. It is used by clients that export U3D data.

Importing DL

The U3D Importing dynamic library contains IFXCOM plug-ins and services to decode a U3D data stream. It is used by clients that import U3D data.

Rendering

The U3D Rendering dynamic library contains IFXCOM plug-ins and 3D rendering services (Direct3D* 7/8 and OpenGL*, possibly software in the future). Used by clients needing a pre-built solution to render imported or dynamically generated U3D data.

Scheduling DL

The U3D Scheduling dynamic library contains IFXCOM plug-ins and scheduling services. Used by clients needing a pre-built solution to manage asynchronous U3D data streaming, its decompression and for controlling simulation time.

Note:
The IFXRendering and IFXScheduling DLs are not a part of the U3D specification.

The IFXExtensibility DL is not included here because it is a future dynamic library that does not exist yet. It is expected after the extensible profile is available in a future edition of ECMA-363.
