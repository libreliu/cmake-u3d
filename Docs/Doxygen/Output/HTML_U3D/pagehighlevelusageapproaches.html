High-level Usage Approaches

Some common U3D Sample Software usage activities are described below for exporters and viewers/players.

Exporter Activities

Exporters will typically perform the following steps:

  1. Initialize and setup the system
    • Component system
    • Core Services
    • Scene Graph
  2. Read source CAD scene and model data, and put into the U3D Sample Software's internal run-time format by
    • Constructing a scene graph
    • Creating and setting up author-meshes and other geometry components such as line sets, point sets and 2D glyphs
    • Creating corresponding material and shader definitions
    • Alternative available to generate intermediate data text format (IDTF) files and invoke example U3D converter
  3. Compile each of the author-meshes to generate Continuous Level of Detail (CLOD) author-meshes
    • Define what mesh clean-up or scrubbing operations are performed
  4. Use the Write Manager to output a U3D file based on the current run-time state and the desired configuration
    • Quality levels for different attributes
    • Minimum resolution
    • Streaming priority for various resource
    • Which resource types to include in the generated U3D file

Additional details are described in How To Load & Write U3D and How To Make A CLOD Mesh. The example applications also include a simple exporter in the Hello World style, an intermediate data text format to U3D Converter and a 2D glyph creation demonstration using TrueType.

Viewer/Player Activities

Viewers/players will typically perform the following steps (a subset of these are also done by importers):

  1. Initialize and setup the system
    • Component system
    • Core Services
    • Scene Graph
    • Rendering Device
    • Renderers
  2. Use Load Manager to read U3D file(s)
    • Automatically instantiates palette resources (essentially the scene database) and restores aspects of the run-time environment that were selected in the files for serialization
    • Multiple simultaneous file loads may be performed, synchronously or asynchronously
      • May be performed even after rendering has begun
      • Asynchronous loads require periodic execution of Load Manager tasks
      • Done by IFXScheduling DL if available using the Simulator’s System Manager, or clients may periodically execute the Load Manager
  3. Manually perform any desired additional programmatic creation of geometry, scene graph nodes or other types of resources
  4. Perform scripted or interactive scene modifications and animation by using Scheduling subsystem
    • Facilitated by Simulation Manager, built-in or custom Simulator Tasks, and Time-based Tasks
    • Notification Manager is another important ingredient
  5. Render frames using the Device which is configured to render one or more Scene Graph View Nodes
    • Performs culling and executes modifier chains as needed
    • Shaders then consume and draw the renderable elements

Additional details are described in How To Load & Write U3D. The example applications also include a basic Player and a 2D glyph creation demonstration using TrueType.
