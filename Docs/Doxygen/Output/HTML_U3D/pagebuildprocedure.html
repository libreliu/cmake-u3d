Build Procedure

The U3D Sample Software can be built in two ways, either by using an IDE or fully automated by using the Bash command-line shell.

Microsoft* Visual Studio* .NET* 2003 IDE

To enable usage of the Microsoft* Visual Studio* .NET* 2003 IDE, solution and project files can be generated by performing the following instructions:

  1. Make sure that your INCLUDE, LIB and PATH environment variables are fully defined for your compiler and development environment. For Microsoft* Visual Studio* .NET* 2003, the proper additional settings for these are defined by the VSVARS32.bat batch file whose typical location is in the C:\Program Files\Microsoft Visual Studio .NET 2003\Common7\Tools directory.
    • Refer to the CygwinWithVisualStudioSetup.txt file for details about an easy way to automate the definition of the INCLUDE, LIB and PATH environment variables.
    • Another tool that must also be in the path is uuidgen.exe. The VSVARS32.bat batch file handles this as well.
  2. Run Cygwin* by either using the Cygwin.bat or the CygwinWithVisualStudioSetup.bat batch file located in the directory where it was installed. This is usually C:\Cygwin.
  3. After running Cygwin*, the current directory should be the Source\Tools\BuildSystem directory. If not, then use the necessary cd change directory commands to get there.
  4. Execute the Bash script called vs7_convert.bat. This can be done by typing ./vs7_convert.bat.
  5. Afterwards, the solution file called U3D.sln can be loaded from the IDE and built. It should be located in the Source\RTL\Build\All directory.
  6. To build the example IDTF exporter plug-in for Discreet* 3ds max* 7 and the related setup steps were completed, a stand-alone Exporter.sln solution file can be loaded and built. It is located in the Source\Samples\IntermediateFormat\Make directory. Alternatively, the associated Exporter.vcproj project file can be manually added to the main U3D solution.

Bash Command-Line Shell

To perform a Bash command-line shell based fully automated build perform the following instructions:

  1. Make sure that your INCLUDE, LIB and PATH environment variables are fully defined for your compiler and development environment. For Microsoft* Visual Studio* .NET* 2003, the proper additional settings for these are defined by the VSVARS32.bat batch file whose typical location is in the C:\Program Files\Microsoft Visual Studio .NET 2003\Common7\Tools directory.
    • Refer to the CygwinWithVisualStudioSetup.txt file for details about an easy way to automate the definition of the INCLUDE, LIB and PATH environment variables.
    • Another tool that must also be in the path is uuidgen.exe. The VSVARS32.bat batch file handles this as well.
  2. Run Cygwin* by either using the Cygwin.bat or the CygwinWithVisualStudioSetup.bat batch file located in the directory where it was installed. This is usually C:\Cygwin.
  3. Execute the Bash script called build.bat and specify the desired build system commands. This can be done by typing ./build.sh followed by debug or release to compile either the debug or release build respectively.

The basic build system options, of which only one can be specified, include:

  • debug [XML project name prefix] - Generate debug build for the specified project or all default projects. Default.
  • release [XML project name prefix] - Generate release build for the specified project or all default projects.
  • clean [XML project name prefix] [debug|release] - Clean intermediate and output files for the specified project or all default projects, for only the specified build or all builds.

The additional build system options, of which multiple may be specified, include:

  • --verbose - Generate detailed output that contains all compile-time and link-time info.
  • --log filename - Saves output to a text file of the specified name called filename.

The Bash command-line shell based build will not build the HelloWorld sample application or the Glyph 2d modifier example that uses TrueType fonts. It is necessary to use the Microsoft* Visual Studio* .NET* 2003 based build to create them.

Generated Files

The target binary files generated by the Microsoft* Visual Studio* .NET* 2003 IDE are located in various subdirectories of Source\RTL\Build\All. Those generated by the Bash command-line build system are located in various subdirectories of Source\RTL\Build (not including the All subdirectory).

Whenever a new target binary file is regenerated, it is also copied to a common execution directory Source\U3D. The Bin SDK-like directory contains a duplicate of the target binary files. Beware, all types of builds overwrite the target binary files in this area.

A list of the current target binary files for the Win32* platform are provided in the Software Contents section.

Note:
If recompiling the software, it is currently necessary to place the unzipped package in a directory with a path that contains no SPACE characters. For example, C:\Software\U3D is a compatible path while C:\Documents and Settings\jdoe\My Documents\U3D is not and results in build system failures.
