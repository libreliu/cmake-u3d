Detailed Description

This is the interface to an object that encapsulates all settings of a texture unit (or texture stage).

It is used to pass in these settings to an IFXRender object using IFXRenderContext::SetTextureUnit(). Specific texture units can be enabled or disabled with IFXRenderContext::Enable(IFX_TEXUNITn) or IFXRenderContext::Disable(IFX_TEXUNITn) where n is between 0 and IFX_MAX_TEXUNITS. IFX_TEXUNITn is guaranteed to be equivalant to (IFX_TEXUNIT0 + n).

Each texture unit can be thought of a two separate blending stages, one for RGB and one for Alpha. Each blending stage can have two inputs, one output.

See also:
IFXRenderContext::SetTextureUnit(), IFXRenderContext::Enable(), IFXRenderContext::Disable()

IFXINLINE void IFXRenderTexUnit::InitData (   ) 
 

Initializes all state settings to default values.

Default values are:

  • RGBInput0: IFX_TEXTURE
  • RGBInput1: IFX_INCOMING
  • RGBInterpolator: IFX_CONSTANT
  • RGB Blend Func: IFX_MODULATE
  • Alpha Input0: IFX_TEXTURE
  • Alpha Input1: IFX_INCOMING
  • Alpha Interpolator: IFX_CONSTANT
  • Alpha Blend Func: IFX_MODULATE
  • Tex Coord Generation: IFX_NONE
  • Texure ID: 1 (Default white texture)
  • Texture Transform: Identity matrix
  • Texture Coordinate Set: -1

Reimplemented in CIFXDeviceTexUnit, CIFXDeviceTexUnitDX7, CIFXDeviceTexUnitDX8, CIFXDeviceTexUnitNULL, and CIFXDeviceTexUnitOGL.

IFXINLINE IFXRESULT IFXRenderTexUnit::SetAlphaBlendFunc (  IFXenum  eFunc  ) 
 

Specifies the blending function between AlphaInput0 and AlphaInpu1.

Note: Not all video cards support all blending functions. In this case, the closest approximate match will be used.

Parameters:
eFunc Input IFXenum specifying which blending function to use. Color or alpha values represented below are in the range (0,1). Valid options are:
  • IFX_SELECT_ARG0: out = Input0
  • IFX_MODULATE: out = Input0 * Input1
  • IFX_MODULATE2X: out = Input0 * Input1 * 2
  • IFX_MODULATE4X: out = Input0 * Input1 * 4
  • IFX_ADD: out = Input0 + Input1
  • IFX_ADDSIGNED: out = Input0 + Input1 - 0.5
  • IFX_ADDSIGNED2X: out = (Input0 + Input1 - 0.5) * 2
  • IFX_INTERPOLATE: out = Input0*Diffuse.Alpha + (Input1*(1-Interpolator.Alpha))
  • IFX_DOTPRODUCT3: out = Input0.R*Input1.R + Input0.B*Input1.B + Input0.G*Input1.G. When this is specified, the RGBBlendFunc must be IFX_DOTPRODUCT3. If IFX_DOTPRODUCT3 is not supported (or the RGBBlendFunc is not IFX_DOTPRODUCT3), then IFX_SELECT_ARG0 is used instead.
Returns:
An IFXRESULT error code.
Return values:
IFX_OK No error.
IFX_E_INVALID_RANGE The value specified is invalid.

IFXINLINE IFXRESULT IFXRenderTexUnit::SetAlphaInput0 (  IFXenum  eInput  ) 
 

Specifies input 0 for the Alpha blending stage.

Note: not all video cards will work with every available input. In those cases, the closest match is used.

Parameters:
eInput Input IFXenum specifying the input to be used. Valid values are:
  • IFX_INCOMING: Use the output of the previous texture unit. This is the same is IFX_DIFFUSE for IFX_TEXUNIT0.
  • IFX_DIFFUSE: Use the lit diffuse material alpha.
  • IFX_TEXTURE: Use the alpha value retrieved from the associated texture.
  • IFX_CONSTANT: Use the alpha value specified with IFXRenderContext::SetConstantColor().
All of these options can be combined with IFX_INVERSE. For example (IFX_DIFFUSE | IFX_INVERSE) will use the result of (White - DiffuseAlpha) as the actual input.

All of these options are implicitly combined with IFX_ALPHA. It is not possible to use color values as input into the Alpha combiner.

Returns:
An IFXRESULT error code
Return values:
IFX_OK No error.
IFX_E_INVALID_RANGE The value specified is invalid.

IFXINLINE IFXRESULT IFXRenderTexUnit::SetTexCoordGen (  IFXenum  eGenMode  ) 
 

Specifies the texture coordinate generation function for this texture unit.

Parameters:
eGenMode Input IFXenum specifying the texture coordinate generation function. Valid values are:
  • IFX_NONE: The texture transform is applied to incoming texture coordinates.
  • IFX_TEXGEN_VIEWPOSITION: The vertex position transformed into eyespace is used as the texture coordinate, then the texture transform is applied.
  • IFX_TEXGEN_VIEWNORMAL: The vertex normal transformed into eyespace is used as the texture coordinate, then the texture transform is applied.
  • IFX_TEXGEN_WORLDPOSITION: The vertex position transformed into worldspace is used as the texture coordinate, then the texture transform is applied.
  • IFX_TEXGEN_WORLDNORMAL: The vertex normal transformed into worldspace is used as the texture coordinate, then the texture transform is applied.
  • IFX_TEXGEN_REFLECTION_CUBE: The reflection vector in eye space is used as the texture coordinate, then the texture transform is applied. This is meant to be used as input into a cube map.
  • IFX_TEXGEN_REFLECTION_SPHERE: The eye space position and normal is used to calculate the u,v coordinates into a 2D spherical reflection map, then the texture transform is applied.
Returns:
An IFXRESULT error code.
Return values:
IFX_OK No error.
IFX_E_INVALID_RANGE The value specified is invalid.

IFXINLINE IFXRESULT IFXRenderTexUnit::SetTextureCoordinateSet (  I32  iTexCoordSet  ) 
 

Sets the texture coordinate set to use for applying textures.

If the mesh does not contain texture coordinates for this index, then the first texture coordinate set (index 0) is used. If -1 is specified, then the texture coordinate set used is dependent on the tex unit number of this texture unit. For example, IFX_TEXUNIT7 would try to use the 8th set of texture coordinates (index 7).

Parameters:
iTexCoordSet Input index of the texture coordinate set to use for this texture unit.
Returns:
An IFXRESULT error code.
Return values:
IFX_OK No error.
IFX_E_INVALID_RANGE The iTexCoordSet parameters was not in the range [-1, 7].
