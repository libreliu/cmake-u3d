IFXRESULT CIFXTaskManager::AddTask (  IFXTask *  pTask,
IFXTaskData *  pData,
U32  priority,
IFXTaskHandle *  pTaskHandle
)  [virtual]
 

This method adds a new task to the task manager.

Internally this creates an IFXTaskManagerNode to hold the registration data. A task handle is returned which identifies the registration, and can be used to remove the task from the TaskManager.

The Task Data is a template of the event this task is to handle. Fields which are filled with default values (zero, null, etc) are treated as wild fields and match any event; fields with a specific value must match the task data associated with a particular event in order for the IFXTask to be invoked. The registered IFXTaskData is not actually passed to the IFXTask::Execute method; rather, the IFXTaskData that is associated with the event is passed to the Execute method.

Parameters:
pTask The IFXTask pointer.
pData The IFXTaskData pointer.
priority The priority for this task. Priority is used to resolve the order of execution when multiple tasks are registered to handle the same event.
pTaskHandle Returns the task handle.
Returns:
Returns an IFXRESULT code.
  • IFX_OK upon success.
  • IFX_E_NOT_INITIALIZED if the component has not been initialized.
  • IFX_E_INVALID_POINTER is returned if pTask, pData, or pTaskHandle is NULL.

IFXRESULT CIFXTaskManager::CreateView (  IFXTaskData *  pPattern,
IFXTaskManagerView **  ppView
)  [virtual]
 

This method creates an IFXTaskManagerView for a given event.

An event is defined by an IFXTaskData object. This method matches the event to the IFXTaskData "pattern" contained in each registered IFXTaskManagerNode. Each match results in the node pointer being added to the IFXTaskManagerView. Thus, the View contains all matching task handlers for a given event. Since tasks can be added, removed, and modified at virtually any time, a given view is highly time sensitive and should generally not be held.

It is possible for a View to contain no data (i.e., no task nodes). This simply indicates that there were no matching tasks.

See IFXTaskManagerView for more information.

Parameters:
pPattern The IFXTaskData which defines the event.
ppView Returns an IFXTaskManagerView. ppView must not be NULL -- the caller must supply a valid pointer - but the pointer should not actually contain a IFXTaskManagerView. Rather, an IFXTaskManagerView will be created and returned when this method is called, and the contents of ppView will be overwritten.
Note:
For efficiency, the Task Manager recycles TaskManagerViews. It will not recycle a view object as long as anyone else has a pointer to it, and uses the reference count to verify this. For this reason it is very important not to keep a non-reference-counted pointer to a returned IFXTaskManagerView, as the contents could change without notice.
Returns:
Returns an IFXRESULT code. This may also return errors from IFXCreateComponent if it is necessary to create a new IFXTaskManagerView. It may also return errors from IFXTaskManagerNode::MatchPattern
  • IFX_OK upon success.
  • IFX_E_NOT_INITIALIZED if the component has not been initialized.
  • IFX_E_INVALID_POINTER is returned if pPattern is NULL.
  • IFX_E_INVALID_HANDLE is returned if ppView is NULL.

Implements IFXTaskManager.

This allows modification of registration data.

Returns:
